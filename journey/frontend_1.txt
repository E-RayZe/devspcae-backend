# 🔥 DEVSPACE FRONTEND – DEEP DIVE (REACT NATIVE + EXPO)

## 1️⃣ FRONTEND KA ROLE KYA HAI?

DevSpace ka frontend:

* User ke **saath interact** karta hai
* Backend se **data bhejta / leta** hai
* Decide karta hai **kaun si screen kab dikhani hai**

👉 Frontend **logic nahi**,
👉 Frontend **experience banata hai**

---

## 2️⃣ TECH STACK (WHY THIS CHOICE)

### 🧩 React Native (with Expo)

**React Native** → ek codebase se Android / iOS
**Expo** → setup + build easy

**Why?**

* Fast development
* No heavy native config
* Beginners + production dono ke liye best

---

### 🧩 Expo Router

* File-based navigation
* Web-jaisa routing system

**Why?**

* Screens ka structure clear
* Tabs / Stack easily manage

---

### 🧩 Hooks (`useState`)

* UI ke data ko manage karne ke liye

**Why?**

* Simple
* Functional programming
* Industry standard

---

## 3️⃣ PROJECT STRUCTURE (VERY IMPORTANT)

```
app/
│
├── (tabs)/
│   ├── index.tsx     → Home screen
│   ├── explore.tsx   → Search / Feed
│   ├── chat.tsx      → Chat
│   └── profile.tsx   → Profile
│
├── login.tsx         → Login screen
├── signup.tsx        → Signup screen
├── _layout.tsx       → Navigation config
│
components/           → Reusable UI
hooks/                → Custom hooks
constants/            → Colors, themes
```

👉 Ye structure **real apps jaisa** hai
👉 Viva me bolo: *“File-based navigation using Expo Router”*

## 5️⃣ LOGIN SCREEN (DEEP LOGIC)

### login.tsx me kya hota hai?

#### 🔹 State Management

```js
const [email, setEmail] = useState('');
const [password, setPassword] = useState('');
```

👉 User jo type karta hai, state me store hota hai

---

#### 🔹 Input fields

```jsx
<TextInput
  value={email}
  onChangeText={setEmail}
/>
```

👉 UI ↔ State connected (two-way binding)

---

#### 🔹 Button press

```js
onPress={handleLogin}
```

👉 User action trigger karta hai logic

## 11️⃣ AUTH-BASED FLOW (CONCEPT)

Abhi:

* Login screen manually open

Next:

* Agar token present → Home
* Agar token nahi → Login

👉 Ye **auth-based navigation** kehlata hai
👉 Industry standard pattern

---

## 🔥 FRONTEND ↔ BACKEND COMMUNICATION

Frontend:

```js
fetch('/api/auth/login')
```

Backend:

```js
POST /login → token
```

Token:

* Frontend receive karega
* Store karega (AsyncStorage)
* Future requests me bhejega
